2. jenkins 搭建 P5，P6 的本地连调测试网页，提供给 P5P6 开发人员测试功能
   这里耽误了 2 个小时，因为 jenkins 显示正常部署，但是看网页仍然是老的网页，后来进入服务器查看，发现部署路径错误？但是 jenkins 的配置是完全一致的，为什么会部署路径出错呢？
   原来 P6 项目之前在 ironman 项目，现在移到 activity 项目，2 个项目 build 时候的配置有偏差，一个编译后放在 dist 下，一个放在 dist/static 下，所以不能完全复制 jenkins 的配置。
   但是修正之后还是出错，发现只上传了一个 index.html 文件，js，css 文件夹都没上传，才发现配置路径写成了`dist/*`，但是应该是`dist/**`，这时候百度`linux **和*的区别`，
   发现搜索不到相关内容，经同事提示，才发现应该搜索`linux 一个星号和二个星号的区别`，因为百度搜索会把`*`作为通配符进行搜索。但是解决了这些，发现网页能打开，但是 js 和 css 的链接都是 404，
   才发现还漏处理了 build 时候的引用路径问题，assetsPath，至此，全部问题解决。
